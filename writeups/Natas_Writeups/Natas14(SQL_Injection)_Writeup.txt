#Explanation of the vulnerability(Education Purpose only)
->Suppose you have a website which asks for username and password if look at the source code it looks like this

SELECT * from users where username="" and password="" {This is for explanation purpose}

You see when you enter your username and password it will be placed where the ""(Double quotes are placed) and then it will execute the SQL and in return if there is information present then it will grant you the access and you get logged in, else it will deny the access.

Now  to manipulate or "insert" malicious query you have to understand the SQL language and understand this query

SELECT * from users where username="admin""OR"1"="1 and password="1234" ( You can insert any password that you like)

->When you insert the text in the name field like this admin"OR"1"="1 

*The first thing is that is closes the first quote which is the "admin"
*Then it checks the other condition which is the OR condition OR"1"="1 (The OR Condition means that aleast 1 of the condition is true unlike AND which needs both of them to be true and by logically 1 is equal to 1) This will you give you the access to the account.

Let me write it in clear so you can understand it clearly
